3. loadStudents() - Fetch Data

function loadStudents() {
  fetch(API_URL)
    .then(res => res.json())
    .then(data => {
      const table = document.getElementById("studentTable");
      table.innerHTML = "";
      document.getElementById("totalStudents").innerText = data.length;
      
      data.forEach(s => {
        const row = document.createElement("tr");
        row.innerHTML = `
          <td>${s.id}</td>
          <td>${s.name}</td>
          <td>${s.course}</td>
        `;
        table.appendChild(row);
      });
    });
}

What it does: Fetches students from backend and displays in table

Step-by-step:

  1. Fetch data from API_URL (/students)
  2. Convert response to JSON
  3. Clear existing table rows
  4. Update student count in dashboard
  5. Loop through each student
  6. Create table row for each student
  7. Add row to table

Result: Table populated with student data

4. addStudent() - Create New Record

function addStudent() {
  const name = document.getElementById("name").value;
  const course = document.getElementById("course").value;
  
  if (name === "" || course === "") {
    alert("Please fill all fields");
    return;
  }
  
  fetch(API_URL, {
    method: "POST",
    headers: {"Content-Type": "application/json"},
    body: JSON.stringify({name, course})
  })
    .then(res => res.json())
    .then(() => {
      document.getElementById("name").value = "";
      document.getElementById("course").value = "";
      loadStudents();
    });
}

What it does: Creates a new student record

Step-by-step:

  1. Get form input values (name, course)
  2. Validate: both fields must have values
  3. If validation fails: show alert and stop
  4. Send POST request to backend with student data
  5. Clear form fields
  6. Refresh table to show new student

Result: New student added and table updated

4. GET /students Route - Fetch All Students

app.get("/students", async (req, res) => {
  try {
    if (useDb) {
      const students = await Student.find()
        .sort({_id: 1})
        .lean();
      return res.json(students.map(s => ({
        id: s._id,
        name: s.name,
        course: s.course
      })));
    }
    res.json(fallbackStudents);
  } catch (err) {
    res.status(500).json({error: err.message});
  }
});

What it does: Returns list of all students

Request: GET http://localhost:5001/students

Response:

[
  {id: 1, name: "Rahul", course: "CSE"},
  {id: 2, name: "Priya", course: "IT"}
]

Logic:

  • If useDb is true: query MongoDB
  • If useDb is false: return fallback array
  • Transform MongoDB format to response format
  • Catch errors and return 500 status

5. POST /students Route - Create Student

app.post("/students", async (req, res) => {
  try {
    const {name, course} = req.body;
    if (!name || !course) 
      return res.status(400)
        .json({error: "name and course required"});
    
    if (useDb) {
      const s = new Student({name, course});
      const saved = await s.save();
      return res.json({
        id: saved._id,
        name: saved.name,
        course: saved.course
      });
    }
    
    const student = {
      id: fallbackStudents.length + 1,
      name,
      course
    };
    fallbackStudents.push(student);
    res.json(student);
  } catch (err) {
    res.status(500).json({error: err.message});
  }
});

What it does: Creates a new student

Request: POST http://localhost:5001/students

Request Body:

{
  "name": "Alice",
  "course": "DataScience"
}

Response:

{
  "id": 3,
  "name": "Alice",
  "course": "DataScience"
}

Logic:

  1. Extract name and course from request body
  2. Validate: both fields required
  3. If validation fails: return 400 error
  4. If useDb: save to MongoDB
  5. If not useDb: add to fallback array
  6. Return created student

6. Server Startup

Field Type Required Description

course String Yes Course name (required)

Data Flow Example: Adding a Student

1. User Types "Alice" and "DataScience"
  • Form inputs capture values
2. User Clicks "Add Student"
  • onclick="addStudent()" triggered
  • JavaScript function runs
3. JavaScript Validates
  • Check both fields have values
  • If not: show alert, stop
4. JavaScript Sends Request
  • fetch("/students", {method: "POST", body: {name, course}})
  • HTTP POST request sent over network
5. Backend Receives Request
  • Express route handler intercepts POST
  • req.body contains {name: "Alice", course: "DataScience"}
6. Backend Validates
  • Check required fields present
  • If not: return 400 error
7. Backend Saves Data
  • If MongoDB available: use Student.save()
  • If not: add to fallback array
8. Backend Responds
  • JSON response: {id: 3, name: "Alice", course: "DataScience"}
  • HTTP 200 OK status
9. JavaScript Processes Response
  • Clear form fields
  • Call loadStudents()
10. Table Updates
  • Fetch all students
  • Display in table with new student added
